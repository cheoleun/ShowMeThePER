# 데이터 수집 전략

## 목표

ShowMeThePER의 v1 데이터 수집은 공식 공개 데이터를 우선 사용한다. 상장기업 목록은 KRX 상장종목정보를 기준으로 만들고, 재무제표는 OpenDART 정기보고서 재무정보를 기준으로 수집한다.

v1의 목표는 다음과 같다.

- KOSPI, KOSDAQ 상장기업 목록을 안정적으로 확보한다.
- 각 상장기업을 OpenDART의 `corp_code`와 매칭한다.
- 최근 10년 이상 분석에 필요한 매출, 영업이익, 순이익 데이터를 확보한다.
- 데이터 부족 여부를 숨기지 않고 `coverage`로 기록한다.
- 이후 DB 저장 구조와 성장률 계산 로직을 일관되게 설계할 수 있는 원천 데이터를 만든다.

## 상장기업 목록 확보

상장기업 목록의 1차 소스는 공공데이터포털의 금융위원회 KRX 상장종목정보 API를 사용한다.

- API: `https://apis.data.go.kr/1160100/service/GetKrxListedInfoService/getItemInfo`
- 주요 필드: 기준일, 단축코드, ISIN, 시장구분, 종목명, 법인등록번호, 법인명
- 대상 시장: KOSPI, KOSDAQ
- 갱신 기준: 기준일 다음 영업일 이후 반영되는 데이터를 기준으로 사용한다.

KRX 상장종목정보는 기업이 아니라 종목 단위 데이터다. 우선주, 종류주식, 복수 상장 증권을 안전하게 다루기 위해 내부 모델은 기업과 상장 종목을 분리한다.

- `company`: 기업 단위 정보
- `listing`: 종목코드, ISIN, 시장구분, 상장 상태 등 증권 단위 정보

## OpenDART 기업 고유번호 매칭

OpenDART 재무제표 API는 기업 식별자로 `corp_code`를 사용한다. 따라서 KRX 상장종목정보와 OpenDART 고유번호 목록을 매칭해 내부 기업 마스터를 만든다.

OpenDART 고유번호 API에서 다음 필드를 사용한다.

- `corp_code`: OpenDART 기업 고유번호
- `corp_name`: 기업명
- `stock_code`: 상장기업 종목코드
- `modify_date`: 변경일

매칭 기본 규칙은 다음과 같다.

- 1차 매칭: OpenDART `stock_code`와 KRX 단축코드를 비교한다.
- 2차 검증: 종목코드 매칭 실패 또는 중복 후보가 있으면 기업명과 법인등록번호 기반 후보를 따로 검토한다.
- 매칭 실패 기업은 수집 대상에서 바로 제외하지 않고, 별도 상태값으로 남겨 원인을 추적한다.

## 재무제표 데이터 확보

재무제표의 1차 소스는 OpenDART 정기보고서 재무정보 API를 사용한다.

대량 수집에는 `fnlttMultiAcnt`를 우선 사용한다.

- 여러 회사의 주요계정을 한 번에 조회할 수 있어 전체 KOSPI/KOSDAQ 수집에 적합하다.
- OpenDART 기준으로 2015년 이후 재무정보를 제공한다.
- 한 번에 조회 가능한 회사 수 제한이 있으므로 수집 작업은 회사 목록을 배치로 나누어 실행한다.

보강 수집에는 `fnlttSinglAcntAll`을 사용한다.

- 주요계정 API에서 필요한 계정이 누락되거나 계정명이 애매한 경우 사용한다.
- 전체 재무제표 계정과 `account_id`를 확인해 매출, 영업이익, 순이익 매핑을 검증한다.

보고서 코드는 다음 값을 사용한다.

- `11013`: 1분기보고서
- `11012`: 반기보고서
- `11014`: 3분기보고서
- `11011`: 사업보고서

재무제표 구분은 연결재무제표를 우선한다.

- 1순위: `CFS` 연결재무제표
- 2순위: `OFS` 개별재무제표
- 연결재무제표가 없는 기업은 개별재무제표로 fallback하고, 어떤 기준을 사용했는지 저장한다.

## 수집 대상 계정

v1에서 성장률 계산에 필요한 핵심 계정은 다음 3개다.

- 매출
- 영업이익
- 순이익

계정명은 기업과 업종에 따라 다르게 표시될 수 있으므로, 구현 단계에서 OpenDART `account_id`, 계정명 후보, 재무제표 구분을 함께 사용해 표준 계정 매핑 규칙을 만든다.

## 분기 데이터 산출 방식

성장률 계산의 기준이 되는 분기 값은 다음 방식으로 산출한다.

- Q1: 1분기보고서 값을 사용한다.
- Q2: 반기보고서의 3개월 값이 있으면 사용하고, 없으면 반기 누적값에서 Q1 값을 차감한다.
- Q3: 3분기보고서의 3개월 값이 있으면 사용하고, 없으면 3분기 누적값에서 반기 누적값을 차감한다.
- Q4: 사업보고서 연간값에서 3분기 누적값을 차감한다.

분기별 원천값을 만든 뒤 다음 지표를 계산한다.

- 분기별 YoY 성장률
- 최근 4분기 합산 기준 YoY 성장률
- 연간 YoY 성장률

## 10년 이상 데이터 확보 전략

OpenDART 재무정보 API는 2015년 이후 데이터를 기준으로 사용한다. 현재 기준으로 연간 데이터는 10년 이상 분석에 필요한 원천 데이터를 대체로 확보할 수 있다.

다만 모든 기업이 동일한 기간의 데이터를 갖는 것은 아니다.

- 최근 상장 기업은 상장 전 데이터가 부족할 수 있다.
- 합병, 분할, 회계기준 변경, 결산월 변경이 있는 기업은 시계열이 끊길 수 있다.
- 최근 4분기 합산 YoY는 계산에 선행 4개 분기와 전년 동일 4개 분기가 필요하므로 초기 구간의 계산 가능 기간이 더 짧다.

따라서 데이터 수집 결과에는 `coverage`를 함께 저장한다.

- 수집 가능 연도 범위
- 수집 가능 분기 범위
- 매출, 영업이익, 순이익별 결측 여부
- 연결재무제표 사용 여부
- fallback 사용 여부
- 성장률 계산 가능 구간

기본 필터에서는 필요한 기간의 데이터가 모두 있는 기업만 통과시키고, 화면에서는 데이터 부족 여부를 명확히 표시한다.

## v1 구현 순서

v1은 다음 순서로 구현한다.

1. KRX 상장종목정보 수집
2. OpenDART 고유번호 목록 수집
3. KRX 종목과 OpenDART `corp_code` 매칭
4. 기업과 상장 종목 마스터 저장
5. 데이터 커버리지 리포트 생성
6. OpenDART 주요 재무계정 수집
7. 누락 계정과 애매한 계정 보강 수집
8. 분기/연간 표준 재무값 생성
   - 사업보고서 `11011`은 연간값으로 사용한다.
   - 1분기 `11013`, 반기 `11012`, 3분기 `11014`, 사업보고서 `11011`의 누적값 차이를 이용해 분기값을 산출한다.
   - 현재 구현은 OpenDART 주요계정의 누적 금액을 기준으로 분기값을 산출하며, 3개월 금액 필드가 별도로 필요한 경우 보강 수집 단계에서 확장한다.
9. YoY 성장률 계산
10. 성장률 필터와 순위 기능 구현
    - 성장률 순위는 선택된 성장률 시계열의 최근 구간 최소 성장률을 기준으로 정렬한다.
    - PER, PBR, ROE는 별도 valuation 입력 JSON을 받아 선택 필터와 순위 기준으로 사용한다.
